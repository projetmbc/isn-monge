Mettre en forme une page HTML

Séparer le fond et la forme

Tout comme un écrivain et un journaliste ne se préoccupent pas de la mise en forme de ce qu'ils écrivent, une bonne pratique, lorsque l'on rédige une page HTML, consiste à ne pas mélanger ce que l'on écrit et comment on le met en forme.

Vouloir à la fois mettre en forme un document tout en le rédigeant est une perte de temps. De plus, avec de bons outils, si l'on a bien séparé le fond et la forme, on peut très vite changer l'apparence d'un site. Un exemple très « vendeur » est le site CSS Zen Garden : chacune des pages présentées à cette adresse utilisent le même code HTML ! En comparant cette page et celle-ci, cela semble dur à croire mais c'est bien le cas. Pour les plus soupçonneux, et ils ont raison, il vous suffit d'ouvrir chacune de ces deux pages avec Firefox, et de faire un clic droit pour afficher un menu où vous pourrez cliquer sur « Code source de la page » afin de voir le code HTML utilisé. Nous allons bientôt comprendre comment ce tour de passe-passe est possible.

Le fond, c'est quoi !

Sur la majorité des navigateurs, ce mot et celui-ci apparaissent par défaut tous les deux en gras. Pourtant les codes HTML correspondants sont différents : on a utilisé <b>ce mot</b> et <strong>celui-ci</strong>. Dans un cas, b fait référence à l'initiale de « bold », soit « gras » en anglais, tandis que « strong » signifie « fort ». Dans un cas, on parle de mise en forme, et dans l'autre on indique un mot fort, important dans la compréhension de ce qui est écrit. L'emploi de <b>ce mot</b> est donc une mauvaise chose !

De même, bien que <i>ce mot</i> et <em>cet autre</em> donnent les mises en forme de ce mot et de cet autre, on utilisera <em>cet autre</em> mettre en avant un mot de façon plus douce qu'avec <strong>...</strong>.

Vous connaissez d'autres balises donnant un sens au contenu comme <h2>Un titre</h2>, ou bien <p>Un paragraphe</p>.

Mettre en forme, d'accord. Mais comment ?

Cette page vue dans l'éditeur intégré de SageMathCloud n'aura pas la même mise en forme que si vous la regardez directement dans un navigateur comme Firefox : attardez vous sur les hyperliens et sur les bouts de codes comme ce bout de code.

À quoi cela est-il dû ? En fait, SageMathCloud utilise une technique qui permet de choisir, en autres choses, la mise en forme de ce qui trouve dans les balises <a href="...">...</a> et <code>...</code>. Ceci se fait via l'utilisation des règles de style CSS. Cet acronyme signifie « Cascading Style Sheet» soit « Feuille de style en cascade» (le terme « cascade » se justifie pour des raisons techniques expliqués dans le document Jupyter associé à cette page).

Pour voir le CSS en action, le plus simple est d'ouvrir cette nouvelle page qui reprend exactement le même code HTML que l'actuelle page mais lui ajoute des règles simples de mise en forme de très mauvais goût (du coup, cette dernière phrase n'a plus de sens une fois le lien précédent ouvert).
